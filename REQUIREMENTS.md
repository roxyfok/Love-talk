## 1. 产品概述

### 1.1 产品定义

| 项目 | 内容 |
|------|------|
| **产品名称** | LOVE TALK |
| **产品定位** | 私密情侣聊天模拟器（单页应用 → 全栈应用） |
| **产品形态** | Web 应用（前端 React + 后端 Node.js + 数据库 PostgreSQL） |
| **核心用户** | 希望拥有私密、可定制的模拟聊天空间的个人用户 |
| **核心价值** | 绝对私密的情感陪伴空间，支持多对话对象、自定义词库、"帮我确认"智能回复 |

### 1.2 产品愿景

LOVE TALK 是一个完全本地化/云端可选的私密聊天应用。用户可创建多个虚拟对话对象，每个对象拥有独立的昵称、状态、回复词库和聊天记录。发送消息后，系统从当前对话对象的词库中智能随机抽取词条进行回复，模拟真实的日常对话体验。

### 1.3 功能边界（明确包含 vs 不包含）

**包含功能：**
- ✅ 多对话对象管理（增删切换）
- ✅ 普通词库（100条预设 + 自定义添加/编辑/删除/批量导入/批量删除/分类管理）
- ✅ "帮我确认"智能类别选择系统（自定义类别 + 词条）
- ✅ 反应时间调节（15-60秒，每5秒一档）
- ✅ 用户注册/登录系统
- ✅ 数据云端存储与同步
- ✅ 端到端隐私保护

**不包含功能：**
- ❌ 真实的双人实时通讯
- ❌ 语音/视频通话
- ❌ 文件/图片传输
- ❌ 表情包系统
- ❌ 社交关系网络

## 2. 用户故事

### 2.1 核心场景

**场景一：日常情感陪伴**
> 作为一名独居的上班族，晚上回到家感到孤单，打开 LOVE TALK 和"亲爱的"聊天，发送"今天好累"，看到对方回复"辛苦啦""抱抱""早点休息"，感到温暖。

**场景二：多角色对话**
> 作为创作者，模拟不同性格的角色对话寻找灵感。创建"温柔的TA"和"活泼的TA"两个对象，分别设置不同词库，切换对话体验不同交流风格。

**场景三："帮我确认"决策**
> 用户发送"周末帮我确认一下吃什么"，弹出食物类别选择框，选择"火锅"，对方回复"火锅"。

## 3. 功能需求详述

### 3.1 对话界面

#### 3.1.1 消息发送
- 底部固定输入框，支持键盘输入
- 点击发送按钮或按 Enter 键发送
- 发送后输入框清空，焦点保留

#### 3.1.2 消息展示
- **己方消息**：右对齐，背景 `#f1d6d6`（80%透明度），圆角 `rounded-2xl rounded-br-md`，最大宽度75%，字号13px
- **对方消息**：左对齐，背景 `#dbe5f1`（80%透明度），圆角 `rounded-2xl rounded-bl-md`，最大宽度75%，字号13px
- **时间戳**：气泡内右下角，HH:MM 格式，10px，60%透明度

#### 3.1.3 "帮我确认"触发机制
- **触发条件**：消息中**包含**"帮我确认"四个字（前后可接其他内容）
- **示例触发**：`帮我确认`、`亲爱的帮我确认`、`帮我确认一下时间`
- **触发后行为**：
  1. 弹出类别选择模态框
  2. 显示当前对话对象的所有"帮我确认"类别
  3. 用户点击选择类别
  4. 关闭模态框，显示"正在输入"动画
  5. 等待反应时间后，从选定类别中随机回复一条
  6. 回复后取消"正在输入"状态

#### 3.1.4 普通回复机制
- **触发条件**：不包含"帮我确认"的消息
- **延迟时间**：由设置的 `replyDelay` 决定（默认30秒，可选15/20/25/30/35/40/45/50/55/60秒）
- **回复数量**：1~3条，随机决定
- **回复间隔**：多条之间间隔2秒
- **词条选择**：从当前对话对象的普通词库中随机抽取，不重复
- **同义处理**：词条中含 `/` 时从多个变体中随机选择一个发送
- **"正在输入"动画**：延迟期间显示三个跳动圆点

#### 3.1.5 空状态
- 大爱心图标（32px，`#e1b2b2`，40%透明度）
- 主文案："与「{对象昵称}」的私密对话"
- 副文案："发送消息后，对方会从词库中随机回复"

### 3.2 多对话对象管理

#### 3.2.1 数据模型
```typescript
interface Contact {
  id: string;              // 唯一标识
  name: string;            // 昵称
  status: string;          // 在线状态文案
  wordBank: WordEntry[];   // 普通回复词库
  confirmCategories: ConfirmCategory[];  // "帮我确认"类别
  messages: Message[];     // 聊天记录
  avatarColor: string;     // 头像渐变色
  replyDelay: number;      // 回复延迟（毫秒）
}
```

#### 3.2.2 联系人标签栏
- **位置**：顶部标题栏下方，横向排列
- **样式**：圆角标签（`rounded-full`），活跃标签高亮
- **内容**：渐变色小头像 + 昵称（最多80px，超出截断）
- **滚动**：超出容器宽度时可横向滚动
- **活跃状态**：`bg-[#e1b2b2]/20` + 边框 `border-[#e1b2b2]/40`
- **非活跃状态**：`bg-[#d1d1d1]/20` + 透明边框 + 悬停效果

#### 3.2.3 创建对话对象
- **入口**：标签栏最右侧圆形「+」按钮
- **交互**：弹出对话框，输入昵称后确认
- **默认状态**：
  - 状态文案："在线"
  - 词库：100条预设词条（带分类）
  - "帮我确认"类别：5个预设类别（时间、地点、人物、心情、食物）
  - 头像色：6种渐变色中随机
  - 反应时间：30秒
- **创建后**：自动切换为新对象

#### 3.2.4 切换对话对象
- **操作**：点击任意非活跃标签
- **效果**：消息区域切换显示该对象的聊天记录

#### 3.2.5 删除对话对象
- **操作**：点击活跃标签右上角「x」按钮
- **确认机制**：
  - 首次点击：按钮高亮，提示"再点一次确认删除"
  - 3秒内再次点击：确认删除
  - 超过3秒未操作：自动取消
- **保护规则**：只剩一个时删除后自动创建默认对象

### 3.3 普通词库系统

#### 3.3.1 预设词条（100条，11个分类）

| 分类 | 数量 | 示例 |
|------|------|------|
| 早安问候 | 12条 | 早安/早上好呀、记得吃早餐/别空腹 |
| 晚安睡前 | 10条 | 晚安/好梦呀、盖好被子/别着凉了 |
| 关心体贴 | 18条 | 吃了吗/吃饭了吗、你就是我的全世界 |
| 撒娇可爱 | 18条 | 亲爱的/宝贝/宝宝、哼/不理你了/生气气 |
| 表达爱意 | 17条 | 我爱你/好爱你、这辈子认定你了 |
| 日常分享 | 13条 | 刚看完一部电影/推荐你看、随便/你决定/听你的 |
| 情绪安慰 | 8条 | 别难过/有我在呢、别生气了/我错了 |
| 暧昧调情 | 4条 | 你在想我吗/有没有想我、想抱着你睡/想被你抱 |
| 时间 | 0条（预留分类） | 用于"帮我确认"联动 |
| 地点 | 0条（预留分类） | 用于"帮我确认"联动 |
| 人物 | 0条（预留分类） | 用于"帮我确认"联动 |

#### 3.3.2 词库操作
- **单条添加**：输入框 + 分类选择下拉框 + 「+」按钮/Enter
- **批量导入**：展开文本区域，每行一条，可选择分类
- **编辑词条**：点击铅笔图标，可修改文本和分类
- **单条删除**：点击「x」图标
- **批量删除**：进入批量模式，勾选复选框，底部操作栏确认删除
- **分类筛选**：顶部分类标签按钮，点击筛选显示
- **分类展示**：词库按分类分组显示，带分类标签

#### 3.3.3 同义表达语法
- 使用 `/` 分隔多个同义表达
- 示例：`亲爱的/宝贝/宝宝` → 随机回复"亲爱的"、"宝贝"或"宝宝"

### 3.4 "帮我确认"系统

#### 3.4.1 预设类别（5个）

| 类别 | 示例词条 |
|------|----------|
| **时间** | 马上、等一下、今晚、明天、后天、这周末、等你有空、晚上八点 |
| **地点** | 你家、我家、咖啡店、公园、商场、电影院、餐厅、海边 |
| **人物** | 就我们俩、带上朋友、叫上你闺蜜、叫我兄弟一起、我一个人来 |
| **心情** | 好开心、有点难过、超激动、好平静、很期待、有点紧张 |
| **食物** | 火锅、烧烤、日料、中餐、甜品、奶茶、披萨、寿司 |

#### 3.4.2 使用流程
```
用户发送消息("周末帮我确认吃什么")
        ↓
检测包含"帮我确认"
        ↓
弹出模态框（显示所有"帮我确认"类别）
        ↓
用户点击选择类别（如"食物"）
        ↓
关闭模态框，显示"正在输入"动画
        ↓
等待 replyDelay 时间
        ↓
从选定类别中随机选取一条回复
        ↓
显示对方消息（如"火锅"）
```

#### 3.4.3 管理功能
- **添加类别**：输入名称创建新类别
- **删除类别**：点击类别标题旁「x」
- **添加词条**：展开类别后逐条添加
- **删除词条**：点击词条标签旁「x」
- **展开/收起**：点击类别标题切换

### 3.5 反应时间设置

#### 3.5.1 可选档位
15秒、20秒、25秒、30秒（默认）、35秒、40秒、45秒、50秒、55秒、60秒

#### 3.5.2 作用范围
- 普通回复延迟
- "帮我确认"回复延迟
- 每个对话对象独立设置

### 3.6 设置管理

#### 3.6.1 设置页内容（从上到下）
1. **对方信息** — 昵称输入框、状态输入框
2. **反应时间** — 10个秒数按钮选择
3. **聊天记录** — 清空当前对象聊天记录（带确认）
4. **回复词库** — 分类筛选、批量删除、单条增删改
5. **"帮我确认"类别管理** — 类别增删、词条增删

## 4. 后端需求（待开发）

### 4.1 技术栈
- **框架**：Node.js + Express
- **数据库**：PostgreSQL
- **ORM**：Prisma
- **认证**：JWT (jsonwebtoken) + bcrypt
- **部署**：Railway/Render

### 4.2 数据库模型（Prisma）

```prisma
model User {
  id            String    @id @default(uuid())
  email         String    @unique
  password_hash String
  created_at    DateTime  @default(now())
  updated_at    DateTime  @updatedAt
  contacts      Contact[]
}

model Contact {
  id                 String   @id @default(uuid())
  user_id            String
  name               String
  status             String   @default("在线")
  word_bank          Json     // WordEntry[]
  confirm_categories Json     // ConfirmCategory[]
  messages           Json     // Message[]
  reply_delay        Int      @default(30000)
  avatar_color       String
  created_at         DateTime @default(now())
  updated_at         DateTime @updatedAt
  
  user User @relation(fields: [user_id], references: [id], onDelete: Cascade)
  
  @@index([user_id])
}
```

### 4.3 API 接口

| 接口 | 方法 | 认证 | 说明 |
|------|------|------|------|
| `/api/auth/register` | POST | 否 | 用户注册 |
| `/api/auth/login` | POST | 否 | 用户登录 |
| `/api/auth/me` | GET | 是 | 获取当前用户 |
| `/api/contacts` | GET | 是 | 获取所有联系人 |
| `/api/contacts` | POST | 是 | 创建联系人 |
| `/api/contacts/:id` | PUT | 是 | 更新联系人 |
| `/api/contacts/:id` | DELETE | 是 | 删除联系人 |

### 4.4 安全要求

| 项目 | 要求 |
|------|------|
| 密码哈希 | bcrypt (cost=12) |
| JWT 访问令牌 | 15分钟有效期 |
| JWT 刷新令牌 | 7天有效期，httpOnly Cookie |
| 登录速率限制 | 5次/分钟 |
| 通用速率限制 | 100次/分钟 |
| CORS | 仅允许特定前端域名 |
| HTTPS | 强制 |
| 输入验证 | 所有接口严格验证 |
